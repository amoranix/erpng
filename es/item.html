Artículo

🇺🇸 View in English
⚠️ Traducción automática con Google Translate. Algunos términos técnicos se mantienen en inglés.

Un artículo es un producto o servicio ofrecido por su empresa.

El término Artículo también se aplica a materias primas o componentes de productos que aún no se han producido (antes de que puedan venderse a los clientes). ErpNg le permite gestionar todo tipo de artículos, como materias primas, subconjuntos, productos terminados, variantes de artículos y artículos de servicio.

ErpNg está optimizado para la gestión detallada de sus ventas y compras. Si estás en servicios, puedes crear un Artículo para cada servicio que ofreces. Completar el Item Master es muy esencial para la implementación exitosa de ErpNg.

Para acceder a la lista de elementos, vaya a: > Inicio > Existencias > Artículos y precios > Artículo

  1. Requisitos previos

Antes de crear y utilizar un artículo, se recomienda crear primero lo siguiente:

  • Grupo de artículos
  • Warehouse
  • Una unidad de medida si es necesario
  1. Cómo crear un artículo
  1. Vaya a la lista de elementos, haga clic en nuevo.
  2. Ingrese un Código de artículo, el nombre se completará automáticamente al igual que el Código de artículo al hacer clic dentro del campo Nombre del artículo.
  3. Seleccione un grupo de artículos.
  4. Ingrese las unidades de stock iniciales y la tasa de venta estándar.
  5. Guardar

2.1 Propiedades del artículo

  • Nombre del artículo: El nombre del artículo es el nombre real de su producto o servicio.
  • Código de artículo: El código de artículo es una forma abreviada para indicar su artículo. Si tiene muy pocos artículos, es recomendable mantener el mismo nombre y código del artículo. Esto ayuda a los nuevos usuarios a reconocer y actualizar los detalles del artículo en todas las transacciones. En caso de que tenga muchos elementos con nombres largos y la lista llegue a cientos, es recomendable codificarlos. Para comprender cómo nombrar los códigos de artículos, consulte Codificación de artículos. También puede generar un código de artículo basado en una Serie de nombres habilitando esta función en Configuración de stock.
  • Grupo de artículos: el grupo de artículos se utiliza para categorizar un artículo según varios criterios, como productos, materias primas, servicios, subconjuntos, consumibles o todos los grupos de artículos. Cree su lista de grupos de artículos predeterminada en Configuración > Grupo de artículos y preseleccione la opción mientras completa los detalles de su nuevo artículo en Grupo de artículos. Los grupos de artículos pueden ser subconjuntos, materias primas, etc., o basarse en su caso de uso empresarial.
  • Unidad de medida predeterminada: Esta es la unidad de medida predeterminada que utilizará para su producto. Podrían ser Nos, Kgs, Metros, etc. Puede almacenar todas las UOM que requerirá su producto en Configuración> Datos maestros > UM. Estos se pueden preseleccionar mientras se completa el Nuevo elemento usando el signo % para que aparezca una ventana emergente de la lista UOM. Visita la página UoM para obtener más detalles

2.2 Opciones al crear un artículo

  • Desactivado: si desactiva un artículo, no podrá seleccionarse en ninguna transacción.
  • Permitir artículo alternativo: a veces, al fabricar un producto terminado, es posible que un material específico no esté disponible. Si marca esta opción, puede crear y seleccionar un elemento alternativo de la lista Alternativa de elementos. Para saber más, visita la página Artículo alternativo.
  • Mantener existencias: Si mantiene existencias de este artículo en su inventario, ErpNg realizará un asiento en el libro mayor de existencias para cada transacción de este artículo. Asegúrese de mantener esta opción sin marcar al crear un artículo sin stock (fabricado bajo pedido/ingeniero) o un servicio.
  • Incluir artículo en fabricación: esto es para artículos de materia prima que se utilizarán para crear productos terminados. Si el artículo es un servicio adicional como "lavado" que se utilizará en el BOM, mantén esta opción sin marcar.
  • Tasa de valoración: Hay dos opciones para mantener la valoración de la acción. FIFO (primero en entrar, primero en salir) y media móvil. Para comprender este tema en detalle, visite Valoración de artículos, FIFO y media móvil.
  • Tasa de venta estándar: al crear un artículo, ingresar un valor para este campo creará automáticamente un Precio del artículo en el backend. No funcionará ingresar un valor después de guardar el artículo. En este caso, el Precio del Artículo se crea a partir de cualquier transacción con el Artículo. El precio al que venderás el artículo. Esto se recuperará en Sales Order y Sales Invoice.
  • Está arreglado Asset: marque esta casilla de verificación si este elemento es una empresa Asset. Consulte el Módulo Asset para saber más.
  • Creación automática de Asset al realizar la compra: si el artículo es un Asset de empresa, marque esta casilla de verificación si desea crear activos automáticamente mientras compra este artículo a través del Ciclo de compra. Consulte la página Asset para saber más.
  • Porcentaje de asignación: esta opción estará disponible solo cuando cree y guarde el artículo. Este es el porcentaje por el cual se le permitirá facturar o entregar en exceso este artículo. Si no está configurado, seleccionará desde Configuración de stock.
  • Cargar una imagen: para cargar una imagen para su ícono que aparecerá en todas las transacciones, guarde el formulario parcialmente completado. Sólo después de guardar el archivo aparecerá el botón "Cambiar" en el icono de Imagen. Haga clic en Cambiar, luego haga clic en Cargar y cargue la imagen.

Para la India:

  • HSN/SAC: Sistema Armonizado de Nomenclatura (HSN) y Código de Contabilidad de Servicios (SAC) para GST. Estos números los define el gobierno y los diferentes artículos se clasifican bajo códigos diferentes. Se pueden agregar nuevos códigos HSN si no están presentes en la lista.
  • Tiene calificación nula o está exento: para un artículo que está bajo GST, pero no se le aplica ningún impuesto. Por ejemplo: cereales.
  • No está cubierto por GST: para un artículo que no está cubierto por GST. Por ejemplo: gasolina.

3.1 Marca y Descripción

  • Marca: si tienes más de una marca, guárdalas en Ventas > Marca y preselecciónalos mientras completas un artículo nuevo.
  • Descripción: Descripción del artículo. El texto del Código del artículo se recuperará de forma predeterminada.

3.2 Códigos de barras

Los códigos de barras se pueden registrar en Artículos para escanearlos rápidamente y agregarlos en transacciones. En la tabla Códigos de barras, puede agregar el código de barras de un artículo para escanear. Hay dos tipos de códigos de barras en ErpNg:

  • EAN: El número de artículo europeo es un número de 13 dígitos. EAN se utiliza internacionalmente y es reconocido por más sistemas POS.
  • UPC: El Código Universal de Producto es un número de 12 dígitos. El UPC generalmente se usa solo en EE. UU. y Canadá.

3.3 Inventario

  • Vida útil en días: esto es para un producto lote. El número de días después de los cuales el lote de productos quedará inutilizable. Por ejemplo, medicamentos.
  • Fin de vida útil: para un solo artículo/producto, la fecha después de la cual quedará completamente inutilizable. Es decir, el artículo quedará inutilizable en transacciones y fabricación. Por ejemplo, usarás cristales de plástico para fabricar artículos durante los próximos 5 años, después de los cuales querrás usar cuentas de plástico.
  • Garantía: Para realizar un seguimiento del período de garantía, es necesario que el Artículo esté serializado. Cuando se entrega este artículo, la fecha de entrega y el período de vencimiento se guardan en el maestro de números de serie. A través del maestro del número de serie, puede realizar un seguimiento del estado de la garantía.

Un período de garantía es un período de tiempo en el que un producto comprado puede devolverse o cambiarse.

  • Ud. de peso: la unidad de medida del artículo. Puede ser números, kilos, etc. La UM de peso que utiliza internamente puede ser diferente de la UM de compra.
  • Peso por unidad: el peso real por unidad del artículo. Ej: 1 kilo de galletas o 10 galletas por paquete.
  • Tipo Material Request predeterminado: cuando crea un nuevo Material Request para este elemento, el campo establecido aquí se seleccionará de forma predeterminada en el nuevo Material Request. Esto también se conoce como "sangría".
  • Método de valoración: seleccione el método de valoración, ya sea FIFO o media móvil. Lea Métodos de valoración de artículos para saber más.
  • Permitir stock negativo: cuando se marca, se permitirá que el artículo se vuelva negativo incluso si el stock negativo está desactivado en Configuración de stock. Esto es útil si no desea habilitar el stock negativo en artículos de alto valor, pero se permite que algunos artículos seleccionados de bajo valor se vuelvan negativos durante unos días.

3.4 Reordenamiento automático

Cuando el stock de un artículo cae por debajo de una determinada cantidad, puede configurar un reorden automático en la sección "Reorden automático". Esto debería estar habilitado en Configuración de stock. Esto generará un Material Request para el artículo. El usuario con roles Gerente de Compras y Gerente de Stock será notificado cuando se cree el Material Request.

  • Check in (grupo): ¿En qué grupo de almacenes comprobar la cantidad del artículo?
  • Solicitud de: en qué almacén almacenar el artículo para realizar el pedido.
  • Nivel de reorden: cuando se alcance esta cantidad, se activará la reorden. El nivel de reorden se puede determinar en función del tiempo de entrega y el consumo diario promedio. Por ejemplo, puede establecer el nivel de pedido de la placa base en 10. Cuando solo quedan 10 placas base en stock, el sistema creará automáticamente un Material Request en su cuenta ErpNg.
  • Cantidad de pedido: el número de unidades que se reordenarán para que la suma del costo de pedido y el costo de mantenimiento sea mínima. La cantidad de nuevo pedido se basa en la "Cantidad mínima de pedido" especificada por el proveedor y muchos otros factores.

Por ejemplo, si el nivel de pedido es de 100 artículos, la cantidad de pedido no necesariamente será de 100 artículos. La cantidad de reorden puede ser mayor o igual que el nivel de reorden. Puede depender del tiempo de entrega, el descuento, el transporte y el consumo diario promedio.

  • Tipo Material Request: El tipo Material Request con el que se reordenará el stock. Esto depende de si compras el artículo, lo fabricas tú mismo o lo transfieres entre Warehouse.

Nota: El Material Request se crea a las 12 de la noche dependiendo del nivel de reorden establecido.

3.5 Unidades de medida múltiples

Puede agregar UM alternativas para un artículo. Si la UM predeterminada en la que vendes es números (NoS) pero la recibes en Kilos, puedes establecer una UM adicional con un factor de conversión adecuado. Por ejemplo, 500 números de tornillos = 1 kilogramo, seleccione Kilogramo/Litro como UOM y establezca el factor de conversión en 500. Para obtener más información sobre cómo vender en diferentes UM, visite esta página.

3.6 Números de serie

Permitir stock negativo se eliminó para artículos en serie/lote desde la versión 15. Por lo tanto, a partir de la versión 15, los usuarios no podrán realizar transacciones de stock negativo para artículos en serie/lote aunque Permitir stock negativo esté habilitado en la configuración de stock.

Con los números de serie, puede realizar un seguimiento de la garantía y las devoluciones. En caso de que el proveedor retire algún artículo individual, el sistema numérico ayuda a rastrear ese artículo. El sistema de numeración también gestiona las fechas de caducidad.

Ten en cuenta que si vendes tus artículos por miles y son muy pequeños, como bolígrafos o gomas de borrar, no es necesario que los serialices.

En ErpNg, deberá mencionar el número de serie en algunos asientos contables. Si su producto no es un artículo de consumo duradero importante, no tiene garantía y no tiene posibilidades de ser retirado del mercado, evite proporcionar números de serie.

3.7 lotes

Permitir stock negativo se eliminó para artículos en serie/lote desde la versión 15. Por lo tanto, a partir de la versión 15, los usuarios no podrán realizar transacciones de stock negativo para artículos en serie/lote aunque Permitir stock negativo esté habilitado en la configuración de stock.

Un conjunto de artículos se puede fabricar en lotes. Esto es útil para mover el lote y asociar una fecha de caducidad a un lote determinado.

  • Tiene número de lote: las opciones para el número de lote, la fecha de vencimiento y la conservación del stock de muestra se revelarán al marcar esta casilla de verificación. No puede activar esto si hay alguna transacción preexistente para este artículo. Si esto está deshabilitado, tendrás que ingresar los números de serie manualmente para cada transacción.
  • Serie de números de lote: prefijo que se aplicará a los números de lote. Si configura 5x1SCR, el primer lote se denominará como 5x1SCR00001 en la primera transacción/fabricación.
  • Crear automáticamente un nuevo lote: si el número de lote no se especifica en las transacciones, se creará automáticamente en un formato como AAAA.00001. Si siempre desea crear manualmente un número de lote para este artículo, deje este campo en blanco. Esta configuración anulará el 'Prefijo de serie de nombres' en la configuración de stock. Los números de lote se pueden configurar para que se generen automáticamente si usted fabrica los artículos, o se pueden ingresar manualmente si provienen de un fabricante externo.
  • Tiene fecha de caducidad: si marca esta opción, el número de lote se creará según la fecha de caducidad. Las fechas de caducidad se pueden configurar en el maestro 'Lote'.
  • Conservar muestra: para conservar una cantidad mínima de existencias de muestra del artículo. Para ello, debe configurar un Warehouse de retención de muestras en la configuración de stock. Para saber más, haga clic aquí.
  • Tiene número de serie: es similar a la serie de números de lote; Se creará cuando realice transacciones/fabricación. Si configura la Serie de números de serie como AA, en la primera transacción se creará un número de serie como AA00001.

Consejo: Al ingresar un código de artículo en una tabla de artículos, si la tabla requiere detalles de inventario, dependiendo de si el artículo ingresado es por lotes o serializado, puede ingresar números de serie o de lote de inmediato en un cuadro de diálogo emergente.

Nota: Una vez que marcas un artículo como serializado, por lotes o ninguno de los dos, no puedes cambiarlo después de haber realizado un Stock Entry.

Para obtener más información, visite la página Conciliación de acciones.

3.8 Variantes

Una variante de artículo es una versión diferente de un artículo. Para obtener más información sobre cómo administrar variantes, consulte Variantes de artículos.

3.9 Valores predeterminados del artículo

En esta sección, puede definir valores predeterminados relacionados con transacciones para toda la empresa para este artículo.

  • Warehouse predeterminado: Este es el Warehouse que se selecciona automáticamente en sus transacciones con este artículo.
  • Lista de precios predeterminada: ya sea venta estándar o compra estándar. Asimismo, también puedes configurar las cuentas predeterminadas de compra y venta
  • Proveedor: si se establece un proveedor predeterminado, este proveedor será seleccionado para nuevas transacciones de compra.
  • Cuenta de Gastos por Defecto: Es la cuenta en la que se debitará el costo del Artículo.
  • Cuenta de Ingresos Predeterminada: Es la cuenta en la que se acreditarán los ingresos por la venta del Artículo.
  • Cost Center predeterminado: se utiliza para realizar un seguimiento de los gastos de este artículo.

Consejo: puedes agregar más filas para varias empresas.

3.10 Detalles de compra y reabastecimiento

  • Unidad de medida de compra predeterminada: la unidad de medida predeterminada que se utilizará en las transacciones de compra.
  • Cantidad mínima de pedido: la cantidad mínima requerida para transacciones de compra como Purchase Order. Si está configurado, el sistema no le permitirá continuar con la transacción de compra si la cantidad del artículo en la transacción de compra es menor que la cantidad establecida en este campo.
  • Existencia de seguridad: “Existencia de seguridad” se utiliza en el informe “Nivel de pedido recomendado por artículo”. Según el stock de seguridad, el consumo diario promedio y el tiempo de entrega, el sistema sugiere el nivel de reorden de un artículo.

Nivel de reorden = Stock de seguridad + (Consumo diario promedio * Plazo de entrega)

  • Tarifa de última compra: aquí se mostrará la tarifa a la que compró este artículo por última vez utilizando un Purchase Invoice.
  • Es un artículo de compra: si no está marcado, no podrá utilizar este artículo en transacciones de compra.
  • El artículo lo proporciona el cliente: se marca si el artículo lo proporciona un cliente y se recibe a través de Stock Entry > Recibo de Material. Si está marcado, el campo Cliente es obligatorio como cliente predeterminado para Material Request. Para obtener más información, visite esta página.
  • Días de plazo de entrega: Los días de plazo de entrega son el número de días entre el pedido del artículo y su llegada al Warehouse.

3.11 Detalles del proveedor

  • Entregado por proveedor (envío directo): si el artículo es entregado directamente por el proveedor al cliente, marque esta casilla de verificación. Lea más aquí.
  • Códigos de proveedor: Seguimiento del código de artículo definido por los proveedores para este artículo. En las transacciones de Compra, al seleccionar un Artículo, también se obtendrá un No. de Parte del Proveedor para referencia del Proveedor. Puede leer más al respecto aquí.

3.12 Detalles del Comercio Exterior

Si obtienes el artículo de otro país, puedes configurar los detalles aquí.

  • País de origen: el país del que obtienes el artículo.
  • Número de arancel aduanero: puede crear un número de arancel aduanero con una descripción y utilizarlo como referencia aquí para compartirlo con agencias de aduanas. Más adelante se podrá utilizar para agregar Delivery Note.

3.13 Detalles de ventas

  • Otorgar comisión: Otorgue una comisión al Vendedor y al Socio de ventas cuando se venda este artículo. Si está deshabilitado, las ventas generadas por este artículo se ignorarán en el cálculo de la comisión.
  • Unidad de medida de ventas predeterminada: la unidad de medida predeterminada que se recuperará para las transacciones de ventas.
  • Descuento máximo (%): Puede definir el descuento máximo en % que se aplicará a un artículo. Por ejemplo: si estableces un 20%, no podrás vender este artículo con un descuento superior al 20%.
  • Es artículo de venta: si no está marcado, no podrá utilizar este artículo en transacciones de venta.

3.14 Ingresos y gastos diferidos

Puede habilitar ingresos o gastos diferidos del artículo. Una vez que marque la casilla de verificación, verá opciones para configurar la Cuenta de gastos diferidos y la cantidad de meses durante los cuales se difieren los ingresos/gastos.

Por ejemplo, considere una membresía anual en un gimnasio: usted paga el dinero por adelantado de una sola vez, pero el servicio se brinda durante todo el año. Para el propietario del gimnasio, esto son ingresos diferidos y para el cliente, es un gasto diferido.

Consulte las páginas sobre Ingresos diferidos para obtener más detalles.

3.15 Detalles del cliente

El Cliente puede identificar un Artículo con un Código de Artículo diferente. esto es similar al Código de proveedor.

  • Nombre del cliente: seleccione un cliente aquí.
  • Grupo de clientes: se obtendrá según el cliente que seleccionó en el campo anterior.
  • Código de referencia: Un cliente puede identificar este artículo con un número diferente. Puede realizar un seguimiento del código de artículo asignado por el cliente para este artículo. Cuando crea un Sales Order, se mostrará el código de referencia del cliente para este artículo.

3.16 Impuesto sobre artículos

Estas configuraciones son necesarias solo si un artículo en particular tiene una tasa impositiva diferente a la tasa definida en la cuenta de impuestos estándar.

Debe crear una nueva 'Plantilla de impuestos sobre artículos' o elegir una existente. Por ejemplo, si tiene una cuenta fiscal, “IVA 14%” y este artículo en particular está exento de impuestos, seleccione “IVA 14%” en la primera columna y establezca “0” como tasa impositiva en la segunda columna. Visite la página Plantilla de impuestos sobre artículos para obtener más detalles.

También puede establecer una categoría de impuestos para este artículo.

3.17 Criterios de inspección

  • Inspección requerida antes de la compra: si es obligatoria una inspección antes de comprar el artículo, es decir, antes de generar Purchase Receipt, marque esta casilla de verificación.
  • Inspección requerida antes de la entrega: si se requiere una inspección en el momento de la entrega por parte de su proveedor para este artículo, marque esta casilla de verificación. Es decir, antes de generar un Delivery Note.
  • Plantilla de inspección de calidad: si se prepara una inspección de calidad para este artículo, esta plantilla de criterios se actualizará automáticamente en la tabla de inspección de calidad de la inspección de calidad. Ejemplos de criterios son: peso, longitud, acabado, etc.

La inspección de calidad se puede realizar con Vista rápida y no es necesario ir a una página diferente para actualizar los detalles de la inspección en ErpNg.

Lea Inspección de calidad para saber más.

3.18 Fabricación

  • BOM predeterminado: el Bill of Materials predeterminado utilizado para fabricar este artículo.
  • Suministro de materias primas para la compra: si está subcontratando a un proveedor, puede optar por proporcionarle las materias primas para fabricar el artículo utilizando el BOM predeterminado.
  • Fabricante: seleccione el fabricante que fabricó este artículo.
  • Número de pieza del fabricante: Introduzca el número de pieza asignado por el fabricante para este artículo.

3.19 Publicación de artículo en sitio web

Desde la versión 16 en adelante, la aplicación Webshop se ha separado del núcleo ErpNg. Para utilizar la funcionalidad de publicar artículos en un sitio web generado desde su cuenta ErpNg, asegúrese de que la aplicación Webshop de Frappe esté instalada en el sitio.

Mostrar en el sitio web: elija si desea mostrar este artículo en su sitio web; haga clic en Acción y elija "Publicar en sitio web".

Se crea un nuevo registro llamado Elemento del sitio web, que contiene detalles adicionales sobre los elementos y también se publica en la vista web del elemento.

Los siguientes campos se encontrarán en el maestro de elementos del sitio web.

*Peso**: los artículos con mayor peso se mostrarán primero en el sitio web. El límite de número que puedes introducir aquí es muy alto.

  • Presentación de diapositivas: se puede mostrar una presentación de diapositivas en la parte superior de la página. Visite la página Página de inicio en el módulo Sitio web para obtener más información.
  • Imagen: puedes adjuntar una imagen en lugar de una presentación de diapositivas.
  • Sitio web Warehouse: seleccione un almacén existente o cree uno nuevo para transacciones en su sitio web. Este Warehouse será diferente de sus Warehouse sin conexión. El stock para cualquier transacción en línea se deducirá del Warehouse establecido en el sitio web Warehouse.
  • Grupos de elementos del sitio web: en esta tabla, puede seleccionar grupos de elementos existentes o crear nuevos para clasificar los elementos de su sitio web.
  • Establecer metaetiquetas: las metaetiquetas ayudan con el SEO. Consulte la Página web para saber cómo agregarlos.

Visite el Fabricación y el Sitio web para comprender estos temas en detalle.

3.20 Especificaciones del sitio web

Esta sección es para configurar otros detalles sobre el artículo.

  • Copiar del grupo de artículos: Los detalles de las 'Especificaciones del sitio web' se recuperarán según lo establecido en un grupo de artículos específico elegido en la sección anterior (2.17).
  • Especificaciones del sitio web: etiqueta y descripción del artículo. Por ejemplo, "Garantía: 1 año".
  • Descripción del sitio web: aparecerá en la página del artículo.
  • Contenido del sitio web: (Introducido en v12) Puede crear estilos adicionales, etc., use el marcado Bootstrap 4 para mostrar en la página del elemento.

3.21 Detalles de publicación del Hub

El centro es un mercado en línea gratuito donde proveedores y clientes pueden realizar transacciones. Si ambas partes están en ErpNg, las transacciones se realizan sin problemas. Puede visitar el centro en: https://hubmarket.org.

  • Publicar en Hub: elija si desea publicar su artículo en https://hubmarket.org/. Es un mercado libre. Si su proveedor/cliente también está en ErpNg, las transacciones se realizarán sin problemas. Por ejemplo, al crear un Purchase Order por su parte, se creará un Sales Order por parte del Proveedor.
  • Hub Warehouse: este es un Warehouse separado para mantener el stock para las transacciones de su centro.
  • Sincronizado con Hub: sincroniza elementos y otros detalles con Hub cuando se realizan transacciones.
